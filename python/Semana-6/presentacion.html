Unidad 1: Sistemas Binarios

La Lingüística Fundamental del Hardware

La relación es total: las computadoras solo entienden "encendido" (1) y "apagado" (0). Todo lo que programamos debe ser traducido a esta secuencia de bits para que la máquina pueda ejecutarlo.

Unidad 2: Lógica

El Cerebro que Toma Decisiones

Si el sistema binario es el "qué" se almacena, la lógica es el "cómo" se actúa sobre ello. Es el conjunto de reglas (AND, OR, NOT) que permite a un software tomar decisiones y crear flujos condicionales.

Aplicación: Flujos Condicionales

Regla: "El usuario debe ser 'Admin' O debe ser 'Editor' Y estar 'Activo'."

Lógica: P OR (Q & R) 

if (rol == "Admin") or \
   (rol == "Editor" and estado == "Activo"):
    
    print("Acceso Concedido")
else:
    print("Acceso Denegado")

Unidad 3: Teoría de Conjuntos

El ADN de las Bases de Datos (SQL)

La teoría de conjuntos es la base matemática para organizar y consultar colecciones de datos. Una tabla en una base de datos es, conceptualmente, un conjunto de registros únicos.

Aplicación: Consultas SQL Ejecutables

Este código crea 2 tablas temporales y muestra las 3 operaciones:

WITH Alumnos_Gimnasia (nombre) AS (
  VALUES ('Ana'), ('Bruno'), ('Carla')
),
Alumnos_Natacion (nombre) AS (
  VALUES ('Bruno'), ('Carla'), ('David')
)

-- 1. UNIÓN (A ∪ B): Todos los alumnos, sin repetir.
-- (Resultado: Ana, Bruno, Carla, David)
SELECT nombre FROM Alumnos_Gimnasia
UNION
SELECT nombre FROM Alumnos_Natacion;

-- 2. INTERSECCIÓN (A ∩ B): Alumnos en AMBAS clases.
-- (Resultado: Bruno, Carla)
-- SELECT nombre FROM Alumnos_Gimnasia
-- INTERSECT
-- SELECT nombre FROM Alumnos_Natacion;

-- 3. DIFERENCIA (A - B): Alumnos SÓLO en Gimnasia.
-- (Resultado: Ana)
-- SELECT nombre FROM Alumnos_Gimnasia
-- EXCEPT
-- SELECT nombre FROM Alumnos_Natacion;

Nota: Para probar las consultas 2 y 3, descomenta sus bloques y comenta las otras consultas.
